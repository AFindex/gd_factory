# 工厂 Demo 说明

## 场景划分

- `scenes/factory_demo.tscn` 仍然是默认主入口，用来保留原本的静态工厂建造与物流 baseline
- `scenes/mobile_factory_demo.tscn` 是新增的概念验证场景，专门演示“移动工厂”在世界中的部署、回收与再部署
- `scenes/mobile_factory_test_scenario.tscn` 是更大的综合测试场景，用来同时观察多台移动工厂、多种尺寸和更复杂的内部物流 case 长时间运行

## 当前版本包含

- 固定俯角的 3D 工厂镜头，支持边界内平移、缩放和分段旋转
- 鼠标驱动的网格指向、建造预览、放置与拆除
- 生产器、可自动形成直线/拐角的传送带、回收站三种基础原型建筑
- 分流器、合并器、跨桥、装载器、卸载器五种物流拓扑件
- 基于固定 Tick 的模拟循环，以及一条开场即可运行的生产演示线
- 传送带的连续物品堆积、阻塞回压和逐段传递
- 一套移动工厂 runtime：site-based 结构宿主、移动工厂 footprint 预留、部署端口桥接，以及可保留内部状态的移动 / 自动部署 / 回收 / 再部署流程
- 一个独立的移动工厂 demo，支持运输态机动、部署预览、观察模式和内部编辑并存
- 一套 profile / preset 驱动的移动工厂 authoring 模型，可描述不同尺寸的移动工厂、不同内部预设和多工厂场景 actor
- 一个更大的移动工厂测试地图：包含玩家工厂、后台巡航工厂、常驻部署工厂、场景地标和多条长期运行的世界侧输出线

## 大型测试场景说明

- 这个场景的目标不是教学，而是回归和观察：打开后就能看到多台移动工厂同时处于部署态和移动态
- 地图范围比 focused demo 更大，并额外放入了北侧扩展区、南侧输出走廊和中央观察脊等地标，方便定位不同试验段
- 不同工厂会使用不同 footprint 尺寸、不同外观配色和不同内部 case preset
- 目前包含的内部 case 侧重于：
  - `基础直通线`
  - `分流回汇`
  - `跨桥中继`
  - `双源汇流`
  - `宽幅缓冲回路`
- 部分后台工厂会保持部署输出，另一些会按固定路线在几个锚点之间移动、部署、回收并重新部署
- 每条长期运行的测试线都配有世界侧回收站，避免长时间挂着后整张图因为没有消费端而彻底堵死

## 移动工厂 Demo 交互

- 默认进入“工厂控制”模式：`W/S` 前进后退，`A/D` 左右转向
- `G` 进入部署预览：鼠标选择落点，`Q/E` 旋转部署朝向，左键确认后移动工厂会自动进场、转向并部署
- `Tab` 切换“观察模式”：这时才会像以前一样用 `WASD / 方向键` 控制固定俯角相机
- `R` 在部署态下回收移动工厂，释放 footprint 与端口预留后回到运输位
- `F` 打开或收起右侧内部编辑分屏；编辑区仍然沿用 `1-4` 选建筑、`Q/E` 旋转、左键放置、右键拆除
- HUD 左上角会显示当前控制模式、生命周期状态、部署提示和模式切换按钮，方便区分“现在在操控工厂还是在观察世界”
- 大型测试场景沿用同样的玩家工厂交互，只是世界中会同时运行其他移动工厂；玩家的操作不会暂停后台工厂

## 当前已知限制

- 这一版仅支持 1x1 建筑
- 传送带目前仍是单通道简化模型，还没有实现异星工厂那种完整双边 lane 逻辑
- 分流器和合并器目前是 1x1 简化拓扑，不是异星工厂原版的双格宽结构
- 装载器/卸载器当前是机器口与带网之间的轻量适配件，不包含机械臂式抓取动作
- 当前代码与中文口径已统一：装载器是“传送带 -> 机器”，卸载器是“机器 -> 传送带”。
- 还没有存档、成长系统、分流物流、机械臂或配方生产
- 输入映射由 demo 场景在运行时注册，暂未完全转到编辑器配置里
- 移动工厂当前只有一个对外输出端口，虽然部署朝向可旋转，但仍主要用于验证机制，不是完整载具系统
- 自动部署目前使用直线 steering 和朝向对齐，不包含复杂地图寻路、避障或多单位协调
- 移动工厂 demo 仍然使用预设好的外部线路，当前重点是验证命令模式、部署流程和输入分流，而不是完整的移动平台建造编辑体验
- 大型测试场景里的后台工厂目前使用确定性的脚本循环，不是通用 AI；它们的意义是保持地图“持续运作”，而不是提供智能对抗

## 冒烟测试

可以用下面的命令跑默认静态 demo 的冒烟测试：

```powershell
& 'D:\Godot\Godot_v4.6.1-stable_mono_win64\Godot_v4.6.1-stable_mono_win64_console.exe' --headless --path 'D:\Godot\projs\net-factory' -- --factory-smoke-test
```

这个测试会验证项目启动、临时放置/拆除，以及默认生产线上的物品送达流程。

可以用下面的命令跑独立移动工厂 demo 的冒烟测试：

```powershell
& 'D:\Godot\Godot_v4.6.1-stable_mono_win64\Godot_v4.6.1-stable_mono_win64_console.exe' --headless --path 'D:\Godot\projs\net-factory' 'res://scenes/mobile_factory_demo.tscn' -- --mobile-factory-smoke-test
```

这个测试会验证：
- 默认会进入工厂控制模式，且观察模式能重新接管相机 WASD
- 运输态下移动工厂能够移动与转向
- 一个无效部署点会被正确拦截
- 面向朝向的 footprint / port 计算会按预期变化
- 移动工厂能自动进场并部署到第一条外部物流线，再持续送达回收站
- 部署态下移动命令会被拒绝
- 回收后 footprint 和端口预留会被正确释放
- 再部署到第二条线路后，同一座移动工厂能够恢复物流输出

可以用下面的命令跑大型综合测试场景的冒烟测试：

```powershell
& 'D:\Godot\Godot_v4.6.1-stable_mono_win64\Godot_v4.6.1-stable_mono_win64_console.exe' --headless --path 'D:\Godot\projs\net-factory' 'res://scenes/mobile_factory_test_scenario.tscn' -- --mobile-factory-large-smoke-test
```

这个测试会验证：
- 新的大型测试场景可以单独加载
- 地图里会同时存在多台移动工厂，且初始状态混合了部署态和移动态
- 地图里至少存在多种移动工厂 profile 和多种内部 preset
- 玩家工厂仍可移动、打开内部编辑区并部署到预设输出线
- 后台工厂在玩家操作期间仍会继续移动或维持输出
- 世界侧回收线会继续累计吞吐，说明长期运行链路仍在工作
