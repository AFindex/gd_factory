# 移动工厂输入接口动画分步重构规划

## 1. 这次重构的目标

输入接口当前的问题不是单个速度参数不对，而是“视觉宿主”和“逻辑阶段”混在一起了：

- 世界传送带上的大包还没形成清晰的交接感，就已经开始播放整套接口动画。
- 同一个大包会在世界侧、桥位、舱内侧、解包舱侧之间出现跳宿主、闪现、补播的问题。
- 逻辑上的缓存位、桥接位、处理位和表现上的锚点没有做到一一对应。

所以这次不从微调入手，而是先把输入接口的动态表现全部清空，回到纯功能基线，然后按阶段逐步恢复。

## 2. 当前基线

本轮代码已经把 `InputPort` 的动态表现关闭，当前应当满足：

- 输入接口仍然保留功能逻辑，世界大包仍可进入移动工厂。
- 输入接口只保留静态建筑模型，不再显示世界侧缓存包、桥接包、舱内缓存包。
- 输入接口不再播放脉冲灯和内部重货动态展示。
- 解包舱仍然保留自己的处理逻辑。

这个基线的目的只有一个：先确认“没有任何输入接口动态演出时，功能链路是稳定的”。

## 3. 重建原则

后续重建时，必须坚持以下原则：

- 一个阶段只允许一个视觉宿主。
- 一个大包在同一时刻只允许出现一次。
- 逻辑缓存和视觉缓存必须一一对应，不能用一个逻辑位驱动两个画面。
- 每恢复一个阶段，只做这一小段，不顺带恢复下一段。
- 每一阶段做完后先由你目测验收，再进入下一阶段。

## 4. 目标链路

最终希望输入接口形成下面这条清晰链路：

1. 世界传送带把大包送到世界侧入口前。
2. 大包从世界传送带自然滑入世界侧入口缓存位。
3. 大包在世界侧缓存位短暂停留，明确表现“接口已接货”。
4. 大包从世界侧缓存位滑到桥位。
5. 大包穿过桥位进入舱内侧缓存位。
6. 大包在舱内侧缓存位等待解包舱接货。
7. 解包舱接货时，大包从舱内缓存位移动到解包接舱位。
8. 大包进入解包处理位，开始处理动画。
9. 处理结束后大包淡出，小包再从解包舱吐出。

## 5. 建议的分步实现顺序

### 第 0 步：纯静态基线

目标：

- 输入接口完全没有动态货物表现。

验收点：

- 不应再看到世界侧、桥位、舱内侧出现任何大包闪现或补播。
- 即使功能仍在流转，输入接口本体也只是一座静态建筑。

### 第 1 步：只恢复“世界传送带 -> 世界侧入口缓存”

目标：

- 只做第一段。
- 大包从世界传送带末端滑进世界侧入口缓存位。
- 一旦进入缓存位，世界传送带上的物体就正式消失。

本步不做：

- 不做桥接。
- 不做舱内显示。
- 不做解包舱联动。

验收点：

- 只能看到一段世界侧接货滑动。
- 大包最终停在世界侧缓存位，不会继续往里走。

### 第 2 步：恢复“世界侧缓存停留”

目标：

- 让大包在世界侧缓存位停留一小段固定时间。
- 这段时间只表达“接口已接货，准备入舱”。

本步不做：

- 不做桥位穿越。

验收点：

- 大包停在世界侧缓存位时，不会闪、不复制、不跳到舱内。

### 第 3 步：只恢复“世界侧缓存 -> 桥位 -> 舱内缓存”

目标：

- 把入舱过程拆成两段连续空间路径：
  - 世界侧缓存位 -> 桥位入口
  - 桥位入口 -> 舱内缓存位
- 这一步只恢复输入接口自己的完整入舱过程。

本步不做：

- 不和解包舱对接。
- 到达舱内缓存后就停住。

验收点：

- 整条路径必须是连续单物体移动。
- 桥位过程里不允许世界侧保留一份，舱内再生成一份。

### 第 4 步：恢复“舱内缓存等待”

目标：

- 大包到达舱内缓存位后，明确停住并等待。
- 舱内缓存位是解包舱接货前的唯一宿主。

验收点：

- 解包舱未准备好时，大包稳定停在舱内缓存位。
- 不会提前跳进解包舱。

### 第 5 步：恢复“舱内缓存 -> 解包接舱位”

目标：

- 当解包舱可接货时，大包从输入接口舱内缓存位移动到解包舱的接舱位。
- 这里只恢复交接，不做处理腔内部动画。

验收点：

- 输入接口的大包离开后，解包舱接舱位才出现该大包。
- 两边不能同时持有同一包。

### 第 6 步：恢复“解包接舱位 -> 解包处理位 -> 淡出”

目标：

- 大包进入解包处理位。
- 处理完成后大包淡出，再开始小包输出。

验收点：

- 大包的消失发生在处理位，而不是接口位。
- 小包输出应当晚于大包淡出，不要重叠得太乱。

## 6. 每一步推荐的实现方式

为了避免再次回到“逻辑和表现纠缠”的状态，建议每一步都按下面顺序做：

1. 先只加一个新的视觉状态枚举，不改其他阶段。
2. 再只加这个阶段对应的唯一锚点。
3. 再只加这段路径的插值。
4. 最后再把状态切换接到逻辑事件上。

不要在同一轮里同时加入多个锚点、多个阶段和多个宿主。

## 7. 输入接口后续推荐的数据结构

为了后面一步步做，输入接口的表现层最好长期保持下面这几个明确的视觉位：

- `WorldApproach`
  - 世界带末端到世界侧缓存位的滑入段。
- `WorldBuffer`
  - 世界侧缓存位。
- `BridgeIn`
  - 从世界侧缓存进入桥位。
- `BridgeOut`
  - 从桥位进入舱内缓存位。
- `InteriorBuffer`
  - 舱内缓存位。
- `UnpackerDock`
  - 解包舱接舱位。
- `UnpackerProcess`
  - 解包处理位。

这几个名字不一定要现在就写死进代码，但概念上必须先固定。

## 8. 推荐的第一步

下一轮建议只做“第 1 步”：

- 世界传送带末端到世界侧缓存位的滑入。
- 先不做缓存停留后的后续动作。
- 先把“接货时机”和“世界侧唯一宿主”彻底做干净。

只要这一步看起来顺了，后面再一段段加，不会再回到现在这种整条链一起坏的状态。
