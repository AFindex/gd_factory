## Context

这次优化横跨世界工厂的建造预览、蓝图应用预览、制造机端口契约和放置交互。当前世界建造预览已经通过 `FactoryPlacement.GetPreviewCenter(...)` 和 footprint 元数据计算多格建筑的预览中心，但蓝图应用预览中的平面覆盖层仍以锚点格为中心绘制固定 `1x1` 方块，因此多格建筑看起来会错位，即使最终 `PlaceStructure(...)` 落地后位置是正确的。

与此同时，`AssemblerStructure` 和 `AmmoAssemblerStructure` 把端口方块直接建进了结构模型，而且它们的三输入三输出端口定义仍是“两侧各两格 + 上下各一格”的混合布局。世界工厂和移动工厂内部编辑器的端口提示覆盖层则共用 `FactoryLogisticsPreview.IsLogisticsKind(...)`，导致除了传送带之外的分流器、合流器、装卸载器、机械臂等预览也会触发这些提示。建造交互本身也只在鼠标按下瞬间尝试一次放置，成功后仍需要重新点击每个格子完成批量搭建。

## Goals / Non-Goals

**Goals:**
- 让蓝图应用预览中的多格建筑使用与最终落地一致的 footprint 中心、朝向和占地范围。
- 重新定义组装机和弹药组装机的 `3x2` 多端口契约，让三个输入与三个输出分别位于 footprint 的两条相对长边上。
- 移除常驻端口方块表现，并把端口提示收敛为“只在传送带放置预览时出现”的上下文覆盖层。
- 让世界工厂的建造模式在成功放置后保持选中状态，并支持按住左键拖拽自动连续放置。
- 保持现有单次点击放置、蓝图提交、库存消耗和无效格拦截逻辑的确定性。

**Non-Goals:**
- 不重新设计所有物流建筑的端口契约，只调整组装机和弹药组装机。
- 不改变蓝图的捕获数据格式或持久化结构。
- 不为删除模式、蓝图框选模式或移动工厂生命周期新增新的拖拽语义。
- 不在本次变更中重做整套建造 HUD，仅优化行为与反馈。

## Decisions

### Use footprint-aware transforms for blueprint preview overlays

蓝图应用预览会继续复用 `FactoryBlueprintPlanner` 产出的计划项，但每个计划项的预览位置和尺寸将不再画成固定锚点方块，而是改为基于该结构的 footprint 计算真实预览中心和预览尺寸。这样世界建造预览、蓝图 ghost 预览和最终放置结果都会共享同一套中心点语义。

之所以优先修正预览覆盖层，而不是改动蓝图 plan 的局部坐标，是因为当前提交路径已经能正确落地多格建筑；问题出在可视化层没有消费 footprint 元数据。

Alternative considered: 调整蓝图捕获与应用的 anchor 归一化规则。该方案被放弃，因为会同时影响已存在的蓝图语义，而当前 bug 并不来自 capture/commit 数据错误。

### Define the `3x2` assembler IO contract on opposite long edges

对于 `3x2` footprint，只有两条长边各自拥有 3 个相邻格，因此三输入三输出要稳定“分别在两边”，必须定义在相对长边上，而不是短边。设计上会把组装机和弹药组装机的输入端口放在一条长边、输出端口放在另一条长边，并由 footprint 旋转逻辑统一派生其他朝向。

这会把端口契约从结构脚本内的局部常量，收敛到共享的 footprint 元数据上，避免视觉、碰撞与物流规则各自维护一份偏移表。

Alternative considered: 继续保留当前混合分布，只微调单个偏移。该方案被放弃，因为无法满足“三进三出分别在两边”的布局目标，也会继续增加玩家判断成本。

### Replace permanent machine port cubes with contextual belt-only overlays

组装机与弹药组装机上的常驻端口小方块会被移除，附近多端口建筑的输入/输出提示将继续通过覆盖层生成，但触发条件会改成“当前处于传送带放置预览”。这让端口提示只在玩家真正进行 belt 对接时出现，而不是在所有物流类建筑预览或平时浏览机器时持续占用视觉注意力。

为了避免世界工厂和移动工厂内部编辑器行为分叉，belt-only 判断应尽量集中在共享的物流预览辅助逻辑或两个调用点的对称条件中。

Alternative considered: 保留结构上的常驻端口方块，只额外收紧覆盖层显示条件。该方案被放弃，因为用户反馈的“方块标记”来自整体表现噪声，单独关闭覆盖层仍会留下常驻视觉干扰。

### Treat placement as a brush while the primary button stays held

世界工厂建造模式会保留当前选中的建造对象，成功放置后不自动退出。额外增加一次“放置笔刷”状态：当左键按下且当前仍在建造模式时，鼠标经过新的可放置格就立即尝试放置。为了避免同一格在一次拖拽中被重复消费或重复建造，需要记录本次按压已经成功处理过的格子，并在松开左键时清空。

这一方案可以保留现有 `TryValidateWorldPlacement(...)`、库存消耗与 topology rebuild 路径，不需要引入新的批量命令或延迟提交机制。

Alternative considered: 只在鼠标移动事件中批量采样一整条线段后一次性放置。该方案被放弃，因为会增加插值与回滚复杂度，而当前需求只要求“拖动时能自动摆放”。

## Risks / Trade-offs

- [组装机端口契约变化可能让现有 demo 摆位失效] -> Mitigation: 同步检查 `FactoryDemo` 中预置产线，必要时调整带子与机器的默认摆放。
- [连续拖拽摆放可能在同一笔刷路径上重复消耗库存] -> Mitigation: 为每次左键按压维护已成功放置格集合，只对首次经过的有效格执行放置。
- [蓝图预览同时存在 footprint 覆盖层和 ghost 预览时可能显得过于拥挤] -> Mitigation: 让 footprint 覆盖层承担占地校验职责，并确保它和 ghost 使用同一中心/尺寸语义，避免错位造成双重影像。
- [移除常驻端口方块后，非 belt 场景下端口感知会减弱] -> Mitigation: 保留 belt 预览时的高对比端口提示，并让端口布局通过 footprint 规则保持稳定、可预测。

## Migration Plan

1. 更新组装机/弹药组装机共享 footprint 的输入输出偏移定义，并移除结构模型中的常驻端口方块。
2. 调整世界工厂与移动工厂内部的端口提示条件，使其仅在传送带预览时生成。
3. 让蓝图应用预览覆盖层按 footprint 中心和尺寸绘制多格结构，并验证旋转后的占地显示。
4. 为世界工厂建造模式加入连续放置与左键拖拽放置状态，保持现有库存与校验逻辑不变。
5. 运行现有 smoke/手动场景，确认蓝图预览、多格制造机物流连接和拖拽建造行为没有回归。

## Open Questions

- 输入端口应落在默认 east 朝向的上侧还是下侧；本次设计只固定“位于相对长边”，实现时需要选择一个默认方向并在 demo 中保持一致。
- 连续拖拽摆放是否只覆盖世界工厂，还是后续也应扩展到 `MobileFactoryDemo` 的内部建造模式；本次先按世界工厂范围设计。
