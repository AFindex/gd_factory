# Windows 开发规则

当你在 Windows 环境下处理这个仓库时，请遵守以下规则：

- 不要一次性重写大文件。
- 优先先创建文件骨架，再分段补充内容。
- 每次编辑保持小而明确，避免超大补丁。
- 对于生成型文件，优先使用模板或脚本生成，不要直接一次输出超长完整内容。
- 如果一次大编辑失败，下一次必须改为更小的分块编辑，而不是继续整文件重写。
- 如果在 Windows 下连续出现写入失败，下一次优先采用更接近 WSL 的分步工作方式。

文件生成规则：

- 对于超过约 200 行的文件，先生成结构。
- 然后按 section、class、function 逐块补充。
- 除非用户明确要求，否则不要直接整文件替换。

编辑规则：

- 保持现有格式、缩进和换行风格不变。
- 避免使用依赖 PowerShell 的超长单行命令来写入大段内容。
- 优先使用安全的小步 patch，或使用可重复执行的生成脚本。

失败兜底规则：

- 如果两次尝试都无法完成大文件写入，就停止整文件生成。
- 改为只使用分块编辑。
- 明确告知用户遇到了 Windows 下的大文件写入限制，但继续推进拆分后的实现。

额外约束：

- 不要为了省步骤而把多个大段内容合并成一次写入。
- 宁可多次小改，也不要一次性生成完整大文件。
- 对于长文档、长配置、大型测试文件，必须分章节、分模块、分用例生成。

完成前检查：

- 如果该文件类型支持校验，先做语法或格式检查。
- 结束时说明这次新建或修改了哪些部分。
