# Script Stats Dashboard

Godot 4 编辑器插件，一键生成项目脚本统计的静态 HTML 报告并在浏览器中打开。

## 功能

- 扫描项目所有 `.cs` / `.gd` 文件
- 统计文件数、总行数、代码/注释/空行、总大小
- 可视化展示：文件类型分布、目录代码量 TOP 10、最大文件 TOP 15、可折叠目录树
- 支持复制最大文件列表到剪贴板

## 使用方式

1. 将本文件夹复制到项目的 `addons/` 目录
2. 在 Godot 编辑器中启用插件：**项目 → 项目设置 → 插件 → Script Stats Dashboard**
3. 点击编辑器顶部工具栏的 **📊 脚本统计** 按钮（或菜单 **项目 → 工具 → 脚本统计仪表盘**）
4. 自动生成报告并用系统默认浏览器打开

## 重要坑点：为什么必须用纯静态 HTML

本插件的报告最终是本地 `file://` 协议的 HTML 文件。浏览器对 `file://` 下的 **JavaScript 动态加载/执行** 有非常严格的安全限制，导致以下常见方案全部会失效：

| 尝试过的方案 | 失败原因 |
|---|---|
| 直接把 JSON 内嵌到 `const DATA = {...}` | 浏览器解析超大单行 JS 时异常，页面空白 |
| `<textarea>` + `JSON.parse()` | `file://` 下 JS 执行环境受安全策略/扩展干扰，数据渲染为空 |
| `fetch('./data.json')` | Chrome/Edge **直接禁止** `file://` 页面 fetch 本地文件（CORS 阻断） |
| `<script src="data.js">` 加载外部 JS | 报错 `Unsafe attempt to load URL from frame`，被浏览器/扩展阻止 |

**最终方案**：在 Godot 中通过 GDScript 直接把所有数据格式化为 **纯静态 HTML 字符串** 写入文件。浏览器打开时完全不需要执行 JS 来获取数据，只解析静态标签即可。因此兼容 Chrome、Edge、Firefox，甚至禁用 JS 的浏览器。

> 页面中的 JS 仅用于"复制信息"按钮，即使 JS 被禁用，所有统计内容依然完整可见。
